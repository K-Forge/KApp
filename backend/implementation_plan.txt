# Backend Implementation Plan - Dynamic Dashboard (Alpha)

## Goal
Create a dynamic dashboard for Students, Professors, and Admins with JWT authentication and DTO-based communication.

## Architecture
- **Framework**: Spring Boot 3.5.7
- **Language**: Java 21
- **Database**: PostgreSQL
- **Security**: Spring Security + JWT (Stateless)
- **Data Transfer**: DTOs (Data Transfer Objects)

## Roles & Features
1.  **Student**:
    -   View enrolled courses.
    -   View pending and submitted assignments.
    -   Submit assignments (upload link/text).
2.  **Professor**:
    -   View teaching courses.
    -   View students in specific courses.
    -   CRUD Assignments (Create, Read, Update, Delete).
    -   Grade submissions.
3.  **Admin**:
    -   Global CRUD for Courses, Professors, Students, and Assignments.

## Implementation Steps

### 1. Dependencies & Database
-   [ ] **Update pom.xml**: Add `jjwt-api`, `jjwt-impl`, `jjwt-jackson`.
-   [ ] **Update init.sql**: Add `assignment` and `submission` tables.

### 2. Security Layer (JWT)
-   [ ] **JwtTokenProvider**: Generate and validate tokens.
-   [ ] **JwtAuthenticationFilter**: Intercept requests, parse token, set SecurityContext.
-   [ ] **SecurityConfig**: Configure filter chain, disable CSRF (stateless), set permissions.
-   [ ] **CustomUserDetailsService**: Load user details from DB (Person/Member).

### 3. Domain & DTOs
-   [ ] **Entities**: `Assignment`, `Submission`. (Ensure `User`/`Role` mapping exists).
-   [ ] **DTOs**:
    -   `LoginRequest`, `JwtResponse`
    -   `AssignmentDTO`, `SubmissionDTO`
    -   `CourseDTO`, `StudentDTO`

### 4. Repositories
-   [ ] `AssignmentRepository`
-   [ ] `SubmissionRepository`
-   [ ] `CourseGroupRepository` (verify existing)
-   [ ] `StudentCourseRepository` (verify existing)

### 5. Services
-   [ ] **AuthService**: Login logic.
-   [ ] **StudentService**: `getEnrolledCourses`, `getAssignments`, `submitAssignment`.
-   [ ] **ProfessorService**: `getTeachingCourses`, `createAssignment`, `gradeSubmission`.
-   [ ] **AdminService**: Generic CRUD methods.

### 6. Controllers
-   [ ] **AuthController**: `/auth/login`
-   [ ] **DashboardController**:
    -   `GET /api/student/dashboard`
    -   `GET /api/professor/dashboard`
-   [ ] **AssignmentController**:
    -   `POST /api/assignments` (Prof)
    -   `GET /api/assignments/course/{groupId}`
    -   `POST /api/assignments/{id}/submit` (Student)

## Notes
-   Skip unit tests for Alpha.
-   Use `Lombok` for boilerplate.
-   Map `employee_role_enum` to `ROLE_PROFESSOR`, `ROLE_ADMIN`.
